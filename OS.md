# [1. OS의 기능](#목차)

## 키워드

### 실행 시

#### BIOS

컴퓨터에 접속된 기기를 제어하는 프로그램들의 모음. OS 기동 전 메모리와 하드웨어를 체크한다. 소프트웨어의 가장 저층에 존재. 일종의 펌웨어다.

#### 부팅 로더

보조기억장치에 저장된 OS를 기동시키기 위한 프로그램.  하드디스크의 MBR에서 실행된다

#### MBR(master boot record)

[하드 디스크](https://terms.naver.com/entry.nhn?docId=824693&ref=y)의 맨 앞에 기록되어 있는 [시스템](https://terms.naver.com/entry.nhn?docId=853283&ref=y) 기동용 영역. [개인용 컴퓨터](https://terms.naver.com/entry.nhn?docId=843096&ref=y)(PC)에 전원을 넣으면 먼저 첫 번째 하드 [디스크](https://terms.naver.com/entry.nhn?docId=821165&ref=y)의 MBR에 기록되어 있는 [프로그램](https://terms.naver.com/entry.nhn?docId=833746&ref=y)이 읽힌다.



### CPU

제어와 연산을 담당하는 컴퓨터 장치


### 커널
OS의 핵심, 사용자의 조작을 처리하여 OS가 작동하도록 한다.
### 쉘
커널과 사용자를 연결하는 UI, 사용자의 조작을 커널에게 전달한다.

기본 기능은 다음과 같음.

- 배치처리
- 히스토리 기능
- 알리아스 기능
- 잡 컨트롤
- 리다이렉트/파이프
- 와일드카드 전개



두 개의 인터페이스로 나뉜다.

- CUI(캐릭터 유저 인터페이스)
- GUI(그래픽 유저 인터페이스): windows 탐색기가 대표적





### 미들웨어
OS와 애플리케이션 사이에서 중개하는 소프트웨어



### 플러그 앤 플레이

디바이스를 컴퓨터에 접속시키면, 드라이버의 설치를 자동으로 수행하는 기능



### I/O 주소 공간

디바이스와의 통신에 사용되는 메모리 영역



### I/O 주소 방식

I/O 주소 공간에 명령, 데이터를 기록함으로써 디바이스를 제어하는 방식



### 주기억장치

레지스터, 캐시, RAM을 의미한다.

#### 레지스터

연산 처리를 위한 CPU 내부의 일시적 기억장치. RAM보다 용량은 작지만 고속으로 작동한다.

#### 캐시

 CPU 칩 안에서 RAM과 CPU간의 속도 차이를 극복하기 위한 저장 공간. 레지스터보다 느리다. 캐시는 CPU 외부, 보조기억 장치에 마련되는 경우도 있다.

#### RAM

캐시보다 느리다. 주기억 장치중에 가장 느리고, CPU칩 외부의 장치중에 제일 빠르다.



### 화면보호기

한 픽셀이 같은 색만을 계속 내보낼 경우 소자가 타버리는 현상(형광 연소)을 막기 위한 기능. 요즘은 CRT를 잘 안 쓰므로 의미 없다..





### 하이퍼네이션

주전원 OFF된 정지 상태





### 컴파일러

어떤 언어의 코드를 다른 언어로 바꿔주는 프로그램. 보통 고수준 언어를 기계어로 번역하는 경우에 한정한다.

#### 트랜스컴파일러

고급 언어간의 번역

#### 디컴파일러

기계어를 고급 언어로

#### 어셈블러

어셈블리어를 기계어로



### 인터프리터

파이썬이나 자바스크립트같은 언어를 한 줄 씩 번역하여 실행하는 프로그램



# [2. CPU&프로세스](#목차)

## [잡과 태스크](# 목차)

### 잡

인간 입장에서이 컴퓨터 작업 단위. 여러 프로그램의 실행이 포함된다.

### 잡 스텝

잡을 작게 나눈 처리 단위



### 태스크

- 잡과 상반되는 개념
- OS 입장에서의 작업 단위. CPU에 처리를 의뢰한다.
- 잡 스텝을 나눈 단위
- [프로세스](###프로세스)나 [스레드](###스레드) 또한 일종의 태스크다.



## [스루풋](#목차)

OS의 단위 시간당 처리 능력



### 턴어라운드 타임

- 명령 입력부터 다음 명령 입력 가능 상태까지의 총 시간.
- CPU의 처리 시간(**리스폰스 타임**)과 입출력 시간으로 이루어진다.



### 스풀(spool)

CPU에서 입출력 장치로 가능 명령을 메모리와 같은 **비교적 고속**인 기억 장치에 일시적으로 저장하는 것.

CPU와 입출력 동작의 속도 차이를 완화한다.



## [인터럽트](#목차)

실행 중인 프로그램을 중단하고 우선순위가 높은 다른 프로그램에 할당을 하는 것.



### 내부 인터럽트(Trap)

실행 중인 프로그램이 원인. 잘못된 명령이나 데이터를 사용하는 경우에도 발생

- 페이지 폴트(무효한 메모리 참조)
- 0으로 나누는 경우
- 오버플로우, 언더플로우
- 잘못된 명령어
- 슈퍼바이저 콜
- 트레이스(디버그를 위해 프로그램을 한 명령씩 실행하는 것)



### 외부 인터럽트

주변 기기의 입출력 동작, 하드웨어 고장, 오작동으로 인한 인터럽트

- 기계 체크 인터럽트
- 타이머 인터럽트
- 입출력 인터럽트
- 콘솔 인터럽트(키보드 조작 등)



## [프로세스](#목차)

- **메모리 상에 할당된 프로그램**. 

- 하나의 프로그램을 여러 개의 프로세스로 만드는 것도 가능함



### 멀티태스킹

- 여러 개의 태스크(혹은 프로세스)가 병행하여 돌리는 것
- 매우 빠르게 전환해가며 동시에 진행하는 것으로 보일 뿐. 실제로는 동시가 아니다
- 태스크 전환에 드는 시간을 **오버헤드**라고 부른다.
- **프리엠티브**: OS가 주도적으로 태스크에 사용권을 부여&박탈하는 방식
- **논프리엠티브**: 태스크가 자발적으로 메모리 점유권을 해제하는 방식. 현재는 별로 안 쓰임



### 시분할 처리 시스템(Time Sharing System)

- 여러 명의 사용자가 한 대의 컴퓨터를 돌아가며 사용하는 것
- 이 또한 마찬가지로 동시에 이용하는 것으로 보이지만, 실은 동시가 아니다.



### 프로세스 상태

- 대기상태: 입출력 동작을 기다리는 상태. 입출력 인터럽트에 의해 "실행 가능 상태"가 됨
- 실행 가능 상태: CPU의 사용 권한 할당을 기다리는 상태. **디스패치**에 의해 실행상태로 바뀜
- 살행 상태: 프로세스에 CPU 사용 권한이 할당되어 실행되는 상태. **프리엠션**으로 '실행가능상태'로 돌아감



### 디스패치

OS가 프로세스에 CPU 사용권한을 할당하는 것



### 프리엠션

OS가 프로세스에 CPU 사용권한을 해제하는 것



## [프로세스 제어](#목차)

### 프로그램 상태어(PSW: program status word)

- 인터럽트로 **중단**된 프로그램의 상태나, 연산의 종료 상태를 저장해 두는 **레지스터**
- 영어단어를 표현할 정도의 작은 용량(64비트)이라서 "word"라고 특이하게 부른다.



### 프로세스 제어 블럭(PCB: process control block)

- 각 프로세스의 상태를 저장해 두는 **메모리**



### 컨텍스트 스위치

프리엠션할 때 PCB에 저장, 디스패치할 때 PCB로부터 복원하는 것



## [스케줄링](#목차)

- 스케줄링을 위한 소프트웨어를 '스케줄러'라고 부른다.

- 스루풋이 향상되고, 응답 시간이 단축되는 스케줄링이 좋은 스케줄링이다.

## [프로세스 분기](#목차)

### fork

- 시스템 콜의 일종. UNIX 게열의 OS에서 이를 통해 프로세스를 복사한다.
- 원본을 부모 프로세스, 사본을 자식 프로세스라고 한다.



### exec

- 자식 프로세스에서 "다른 프로그램을 실행하도록" 커널에 지시하는 시스템 콜



### wait

- 시스템 콜의 일종.
- 자식프로세스 완료 => 부모 프로세스에 시그널 => 부모 프로세스가 커널에 wait 콜 => 커널에 의한 자식프로세스 종료& 부모 프로세스 재개



## [프로세스의 동기](#목차)

### 배타 제어

- 데이터의 무결성을 보존하기 위해, 하나의 프로세스의 처리가 끝날 때까지 자원을 독점시키는 것



### 데드락

- 배타 제어에서 파생되는 문제. 
- 2개의 프로세스가 각자 다른 자원을 독점한 채, 서로의 자원에 액세스하려고 할 때 발생
- 영구적인 대기 상태가 되어버린다.



### 뮤텍스(mutex)

- 데드락을 피하기 위한 방법 1
- 공유 자원에 하나의 프로세스가 액세스 중이면 다른 프로세스는 통행 불가하도록 한다.



### 세마포어(semaphore)

- 데드락을 피하기 위한 방법 2
- 카운터를 사용하여 일정 수 이상의 프로세스의 동시 액세스를 막는다.
- 카운터가 1인 경우가 뮤텍스
- 뮤텍스로도 세마포어를 구현 가능하다.



## [프로세스간 통신](# 목차)

- 프로세스는 각자 다른 메모리 공간을 점유하기 때문에, 원칙적으로 서로를 알 수 없다.
- 프로세스끼리 통신하여 데이터를 주고받는 기능이 몇가지 있다.



### 메시지 큐

- OS의 메시지 기능을 이용한 1대1 통신.



### 파이프

- 부모 프로세스, 자식 프로세스간의 통신을 위한 장치



### 이름 붙은 파이프

- 이름 없는 파이프의 확장판
- 부모, 자식의 관계가 아니더라도 서로 통신이 가능하다.



## [공유메모리와 스레드](#목차)

- 원칙적으로 1 프로세스, 1메모리이지만 프로세스의 **시스템 콜**에 의해 여러 프로세스가 사용하는 메모리 작성 가능



### 어태치

- 공유 메모리에 접속하는 것



### 디태치

- 공유 메모리 접속을 해제하는 것



### 스레드

- 프로세스 안에서의 프로그램 실행의 흐름
- CPU를 사용하는 최소 단위, Light Weight Process라고도 불린다.
- 분기가 가능하여 여러 개의 스레드 동시 실행이 가능하다.(멀티스레드)
- 멀티 프로세스는 여러 메모리에서 일어나고, 멀티 스레드는 한 메모리에서 일어난다.



# [3. 메모리](#목차)

## [메모리 종류](#목차)

### RAM

읽고 쓰기가 가능하지만 전기가 끊기면 휘발된다.

- DRAM
  - 정보를 보존, 유지하기 위해 기억 내용을 다시 쓰는 행위(리프레시)가 필요
  - 저전력
  - 대용량
  - 느린 속도
- SRAM
  - FlipFlop 회로구조. 리프레시 불필요
  - 소비전력 큼
  - 저용량
  - 빠른 속도



### ROM

시스템체크, BIOS 저장 등에 사용된다. 기본적으로 읽기만 가능한 대신 휘발되지 않는다.

냉장고, 밥솥 등 임베디드 기판에 들어가는 메모리도 전부 ROM

- 플래시 메모리: 전기적으로 다시쓰기가 가능한 ROM. USB, SD카드, SSD 드라이브가 여기에 속한다.



## [메모리 공간](#목차)

### 힙

필요한 만큼만 확보(얼로케이션)하여 사용하는 메모리. 필요가 없어지면 원칙적으로는 직접 해제한다. 

### 스택

프로그램 실행 중 이용하는 변수의 내용을 일시적으로 저장하는 메모리. **자동적**으로 확보되고 처리가 종료되면 **자동으로** 해제된다. 스택의 용량은 정해져 있는데 이를 초과하면 스택 오버플로우가 발생한다.





## [가상 기억](#목차)

### 물리 메모리

컴퓨터에 실제 장착된 메모리. OS가 다룰 수 있는 메모리 용량은 정해져 있다. 

(32비트 CPU에서는 최대 2^32byte, 즉 4GB의 메모리까지만 인식할 수 있다.)

이 제한보다 물리메모리가 작으면, 하드디스크의 일부를 메모리로 활용(가상 메모리)하고,

반대로 물리메모리가 더 크면 RAM디스크로 활용한다.



### 가상 메모리

물리 메모리와, 페이징 파일(스왑 파일)이 저장된 디스크 일부를 합쳐 부르는 말



### 스왑

- 스왑 아웃: 물리메모리에 있던, 우선순위가 낮은 프로세스를 디스크(가상 메모리)에 페이징 파일로 저장함
- 스왑 인: 하드디스크에서 메모리로 높은 우선순위의 프로세스를 불러들임

## [메모리 확보와 해체](#목차)

### 얼로케이션(할당)

OS가 프로세스나 데이터의 메모리(힙 영역)을 확보하는 것. 기본적으로 직접 해제해야 한다.



### 리로케이션(재배치)

한 번 확보된 메모리 위치를 변경하는 것.



### 다이나믹 리로케이션(동적 재배치)

프로그램이 실행중인 메모리를 변경하는 것. 스왑이 대표적인 다이나믹 리로케이션이다.



### 가비지 콜렉션

힙 영역은 기본적으로 수동으로 해제해야 하지만, 이를 자동으로 하는 것. 힙 영역이 가득 차서 액세스가 불가능할 때 일어난다.



### 컴팩션

가비지 콜렉션 이후 듬성듬성 흩어진 미사용 영역을 모으는 것. 메모리의 조각모음. 디스크의 조각모음인 디프래그와는 다르다.



## [매핑(페이징, 스래싱, 세그먼테이션, 프래그먼테이션)](# 목차)

OS가 물리 메모리와 가상 메모리 사이에서 프로그램이나 데이터를 대응시키는 것



### 페이징

**4KB의 고정 길이 단위(페이지)**로 매핑을 수행하는 것. 페이징이 빈번한 것을 **스래싱(thrashing)**이라고 한다.

- 페이지 아웃(스왑 아웃)
- 페이지 인(스왑 인)



### 세그먼테이션

**가변 길이 단위**로 매핑을 하는 것. 하나의 프로세스에서 사용하는 메모리는 연속해서 할당한다. 덩어리로 묶이므로 스왑이 적게 일어나는 장점이 있지만, 쓰기 힘든 자투리 메모리가 계속해서 생긴다.(fragmentation) 컴팩션으로 해결해야된다.

- 롤 아웃(스왑 아웃)
- 롤 인(스왑 인)

피자 자르듯이 분할된 영역

### 실린더

### 프래그먼테이션(fragmentation, 단편화)

하나의 파일이 디스크의 여러 트랙에 흩어져있는 상태. 읽는 작업이 느려지는 원인이 됨

### 디프래그(defrag)

연속된 영역으로 매끄럽게 이어주는 작업. 메모리의 [컴팩션](###컴팩션)과 비슷하다.

## [리던던시(redundancy)](#목차)

### 시스템 리던던시

시스템 장애가 발생해도 큰 손실이 일어나지 않도록 여분의 시스템을 갖는 것

- 듀얼 시스템: 동일한 처리를 여러 시스템에서 동시에 돌림. 
- 듀플렉스 시스템: 예비 시스템을 대기시킴. 주 시스템에 장애가 생기면 돌림.



### 데이터 리던던시(패리티 검사, 체크섬)

- 패리티 검사: 0 혹은 1이 등장하는 횟수가 짝수인지, 홀수인지를 한 비트에 저장함
- 체크섬: 데이터의 각 비트의 정보가 모두 반영되어 계산된 산출값을 저장함



### RAID(스트라이핑, 미러링, 해밍 코드, 패리티)

절대 에러가 나서는 안 되는 중요한 시스템 설계에 쓰임 
여러 개의 하드디스크를 연결하여 동시에 액세스. CPU, memory, OS의 부담이 줄어든다.

- RAID0: 데이터를 여러 디스크에 분산하여 기록(스트라이핑). 속도 빠르지만 리던던시 없음
- RAID1: 여러 디스크에 동일한 데이터 기록(미러링). 속도 느리지만 리던던시 O
- RAID2: 각 디스크에 데이터의 해밍코드를 기록한다. 해밍 코드는 패리티의 확장판 격으로, 수정까지 가능케 함
- RAID3: 별도의 디스크에 패리티를 저장함
- RAID4: RAID0 + RAID3
- RAID5: RAID1 + RAID3. 주 데이터와 패리티가 동시에 오류나서 "이상 없음"을 반환할 확률은 극히 적으므로 이를 이용한다. 디스크를 한 개만 이용한다면 오류시 데이터와 패리티 중 무엇이 잘못인지 모르므로 수정이 불가능하지만, "이상이 없는 다른 디스크의 패리티"를 이용하면, 데이터를 수정해야 할지 패리티를 수정해야 할 지 알 수 있다.

### TCP/IP 

패킷 교환 방식을 관장하는 통신 규약. TCP는 데이터를 패킷 단위로 분할한다. IP는 수신처 정보를 붙여서 네트워크로 보낸다. 

OS 레벨에서 TCP/IP 프로토콜을 지원한다.

### HTTP

웹브라우저의 요청에 따라 html, 그림을 주고받기 위한 통신 규약. 애플리케이션 레벨에서 HTTP 프로토콜을 지원한다.



### 기타

- FTP 프로토콜: 컴퓨터끼리 파일을 전송
- Telnet 프로토콜: 넷상의 컴퓨터에 접속하여 CLI로 컴퓨터 조작

- POP3 프로토콜: 메일 서버의 수신함에서 메일을 읽어들임



### OSI

국제 표준화기구에서 정한 **프로토콜 국제 규격**. 서로 다른 기종 간 통신을 수행하기 위함

| 프리젠테이션층 | HTTP, FTP, Telnet, SMTP, POP | 애플리케이션    |
| 세션층         | HTTP, FTP, Telnet, SMTP, POP | 애플리케이션    |

# [6. 여러 OS](#목차)

## [Windows](#목차)

### .NET Framework

MS에서 개발한 Windows 프로그램 개발 및 실행 플랫폼. 여기서 작성된 코드는 컴파일러를 거쳐 CIL(Common Intermediate Language)로 컴파일되고, CLR(Common Language Runtime)이라는 가상의 실행 시스템 상에서 작동한다. (자바 가상머신과 비슷) 따라서 어떠한 CPU나 OS 상에서도 작동 가능하다.



### 특별한 확장자(DLL, bat)

- .DLL(dynamic link library): 각 프로그램에서 사용할 수 있는 처리를 모아둔 파일. 프로그램 실행 중 필요에 따라 기능을 호출하여 이용할 수 있게 해준다.
- .bat(batch file): 명령 프롬프트에서 실행할 명령을 텍스트로 모아 작성한 파일.



### cmd명령어

- md: 디렉토리 만들기(create)
- dir: 현재 디렉토리 표시(read)
- rd: 디렉토리 삭제(delete)
- cd: 디렉토리를 변경
- copy: 지정한 파일을 복사(create)
- type: 지정한 텍스트 파일을 표시(read)
- ren: 지정한 파일의 **이름** 변경(update)
- del: 지정한 파일 삭제(delete)
- cls: 화면을 지움
